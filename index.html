雙層架構設計：notebooklm-cli 與 falo-force 的協同黑魔法

在本專案的架構中，核心技術被拆分為負責底層對話與安全規避的 notebooklm-cli (notebooklm-py)，以及負責上層協作、限制突破與安全審計的 falo-force (Runtime Server & Portal) 平台。兩者分工如下：

維度 notebooklm-cli (底層驅動引擎) falo-force (上層管理平台)
核心定位 Google Web API 協議解析與 TLS/HTTP2 偽裝 多同仁安全協作、資料治理稽核與商用強化網關
解決痛點 解決 Cloudflare WAF 阻斷、無 API Key 對話通信 突破官方單一對話鎖定書庫限制、GAS 雲地派工整合
突破技巧 利用 BoringSSL 覆寫 TLS/JA3，字節級重排 HTTP/2 SETTINGS 幀 動態重組對話歷史並手動注入 (Populate) 客戶端快取，在同一 Session 內切換書庫對話
執行機制 作為獨立 Python 依賴模組，以 cli 工具形式運作 常駐後台 Web 伺服器，採用 Decoupled Subprocess (獨立子行程) 機制動態調用
資料治理 持久化管理 Google 登入 Cookie 與 Session 快取 精密記錄每一發問輪次 (Turn-level) 的同仁 IP、時間、提問書庫名與內容，提供 Log CMS & Excel 導出
部署加值 提供 CLI 指令完成最強帳號登入授權 啟動時自動進行 Port 佔用偵測、自動關閉舊進程，並動態同步 Ngrok 隨機隧道網址至前端配置
