📖 歷史開發研究備忘錄

以下為 pre-2.0 開發階段的研究與技術選型備忘錄，提供給同仁開發參考：

1. NAT / ngrok 連線逾時問題與架構轉型

起因：地端伺服器位於 NAT 內部，使用 ngrok 轉發易因重新連線使外部 URL 變動，且 GAS 呼叫外部 API 有 30 秒執行限制，常造成 Timeout 錯誤。
對策：揚棄傳統雲端直接呼叫地端之架構，轉向「地端主動推送 + 雲端被動接收」的混合輪詢（Hybrid Passive Poll）模式。地端由 gas_adapter.py 定期推送當前網址與主機狀態，雲端僅儲存並呈現該狀態。

2. GAS 模板字串轉義與解析防空處置

起因：直接在 GAS Script 的 HTML 模板字串中嵌入包含換行字元 \n 或反斜線 \ 的變數時，會導致模板渲染引擎解析出錯，造成頁面完全空白。
對策：於資料傳輸時先對特殊字元進行轉義（例如將換行替換為 \\n），並於前端載入時動態解析，確保網頁結構不受損。

3. JavaScript 暫時性死區 (TDZ) 載入阻塞問題

起因：若將 UI 初始化函式（如 initPortal()）宣告於變數或相關 DOM 解析完畢之前，容易觸發 V8 引擎之 TDZ（Temporal Dead Zone）安全機制而阻斷 JavaScript 執行。
對策：優化 JS 載入拓撲結構，確保 initPortal() 的調用置於整個 <script> 區塊的最底部，使所有全域變數與函式已完全宣告。

4. V1 試算表（Spreadsheet）資料表結構參考

歷史版本中使用的核心資料表（Sheets）規劃如下：
• Users：管理系統使用者帳號、密碼及角色權限分流。
• Settings：儲存 API 金鑰、目前地端 URL 及上報狀態指標。
• Folders：映射本地工作目錄與雲端備份目標資料夾。
• Tasks：非同步背景指令與同步任務的狀態監控表。
• Logs：追蹤系統所有存取與操作之稽核軌跡。
• File State：記錄地端檔案雜湊值以供版本比對。
