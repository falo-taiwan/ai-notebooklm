零開埠雙軌 AI 協作引擎

融合地端 Python 原生通訊與瀏覽器外掛的「零開埠、免伺服器 (Zero-Port, Zero-Server)」架構。無痛繞過 WAF 阻擋與內網防火牆限制，開啟雲地協作新次元。

Pure Extension Mode

純外掛與地端 Python (Native Messaging) 連接

此架構免去傳統在本機啟動 WebSocket/HTTP 監聽埠的需求。外掛程式利用 Chrome 底層的系統原生通訊（Native Messaging）機制，直接透過系統管道啟動並調度 Python 進程。

🤖 零 Port 安全通訊

地端 Python 腳本不需要開埠 (No port binding)，免受 Windows/Mac 本地防火牆阻攔及 Port 衝突的困擾。

⚡ 行程自動管理

Python 行程由 Chrome 引擎自動孵化與回收。外掛關閉時，Chrome 會自動中斷 Stdio 管道並殺死進程，防止背景殭屍行程堆積。

HEC Combo Mode

雙分頁監聽組合拳 (HTML Portal + Chrome Extension)

這是一個極為強大的混合架構。外掛 Service Worker 作為訊息調度核心，同時連通「本地控制網頁 (index.html)」與「AI SaaS 官方分頁 (如 ChatGPT/Claude)」，進行零伺服器開銷的雙向數據同步。

👁️ 雙向 DOM 自動化

外掛藉由 content-scripts 同時注入兩個分頁。當 Portal 按下執行，外掛會自動定位 AI 頁面、模擬填入與發送，並以 MutationObserver 抓取輸出回傳。

🛡️ 100% 避開防爬蟲 WAF

由於所有模擬操作都在真實登入的瀏覽器 Tab 中進行，WAF 判定為真實人類操作，免除繞過 Cloudflare 時頻繁出現 403 阻擋的問題。

兩種通訊架構大比拼

評估維度 傳統地端伺服器 (HTTP/WS) 全新零開埠架構 (HEC Native)
開部限制 (Port Binding) 需常駐佔用 Port（如 8765）。容易被防火牆或資安軟體封鎖。 免開 Port。全部在瀏覽器 IPC 及系統 Stdio 中安全執行。
防爬蟲 WAF 繞過 需繁瑣模擬 TLS 指紋（如 JA3）。遇強烈 WAF 時容易遭 403 阻斷。 100% 真人指紋。直接乘載於真實已登入的瀏覽器 Tab 內。
部署便利性 使用者需要安裝複雜 Python 依賴，設定跨域資源共享 (CORS) 策略。 外掛一鍵載入，結合 Native Host 登錄即可運作，實現 Plug & Play。
進程生命週期 地端進程需手動管理。若當機或不正常關閉，可能殘留背景殭屍進程。 由 Chrome 瀏覽器生命週期強制託管。隨瀏覽器關閉自動清理。
