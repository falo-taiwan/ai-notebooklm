AI NotebookLM 區網共享與治理網關設計藍圖

LAN Gateway & AI Governance Architecture Blueprint

1. 核心願景 (Core Vision)

讓區網 (LAN) 內的同仁在無須配置個人 Cookie/API Token 且無須各自登入的前提下，共享託管於伺服器端的「公司最強帳號」與知識庫資料，並確保所有中間操作過程皆可被記錄、稽核與安全防護。

區網同仁 (LAN Users)
透過 Google 試算表 (GAS) 或本地網頁 Portal 送出任務
▼
地端 Python Runtime 治理網關
排隊分發、敏感字詞過濾、身分與行為審計、證據複製
▼
NotebookLM 雲端知識庫 (託管最強帳號)
執行來源上傳、AI 提問與結構化成果生成

2. 系統輸入介面 (Entry Interfaces)

🌐 區網 HTML Portal

伺服器端綁定 0.0.0.0:8765，同仁在相同區網內輸入 http://[伺服器IP]:8765 即可連線。 介面將增設操作者名稱（身分識別）及個人證據備份路徑欄位。

📊 GAS / Google Sheets 中控

雲端共享試算表作為任務派工中心，同仁在上面建立任務、將檔案丟入 Drive 夾。地端 Runtime 定期 Polling 拉取任務並上傳處理，完成後回寫結果連結。

3. 治理與安全機制 (Governance & Security)

  • Who身分審計 (Submitter)：所有調用指令封包強制要求附帶使用者身分參數。
  • What日誌留痕 (Audit Log)：logs/runtime.jsonl 詳盡記錄操作時間、來源 IP、處理之檔案 Hash 及上傳目的地。
  • Evidence證據歸檔 (Evidence Copy)：檔案上傳至 NotebookLM 的同時，地端自動備份原始檔案到 data/evidence/[使用者]/[日期]/，供隨時查驗。
  • Filter資料去識別化 (PII Scrubbing)：前置處理將對文字進行敏感個資與金鑰掃描，阻斷敏感內容送至外網。

4. 雙向 AI 協助 (Pre & Post AI Processing)

1. 前置處理 (Pre-Processing)
透過地端程式或 AI，把複雜的 Excel 表格自動解析為符合 RAG 架構的高可讀性 Markdown 格式，提高 NotebookLM 的知識吸收準確度。
2. 後置處理 (Post-Processing)
如本次實作的 Tab 8 會議記錄轉換器，將 NotebookLM 產出的 Markdown 會議草案自動重構，一鍵載入集團 Word 表格樣板 (`.docx`) 與生成 RWD 網頁檔。

5. 實施階段規劃 (Roadmap)

  1. 第一階段：區網部署與身分註記 (已啟動)：本機 Runtime 跑通、開放區網、並加入 Submitter 註記。
  2. 第二階段：本地過濾防線與證據階層化：設計前置關鍵字掃描與去識別化引擎，並優化 Evidence 歸檔目錄。
  3. 第三階段：試算表自動化下載與分發：健全 GAS 指令包派工、自動下載 Drive 文件、上傳，並回寫完成件連結至 Sheets。
