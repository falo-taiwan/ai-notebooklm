🎯 專案核心使命

本專案旨在打造一個地端優先 (Local-first) 的 AI 代理代理中繼器，串聯使用者地端工作流程與 Google NotebookLM 官方知識庫以及 Gemini 網頁端。透過地端暫存、排隊佇列與 Apps Script (GAS) 的中央上報機制，實現企業內部資訊安全的稽核 governance 與用量統計，並同時提供同仁流暢的網頁介面。

🛠️ 關鍵技術挑戰與解決方案

📐 CSS Grid 循環高度循環依賴

在 grid 子元素強制設為 height: 100% 時，若父容器本身以 min-height 等彈性方式定位，瀏覽器在計算百分比時會因為高度循環參考而將高度降為 natural content height，導致列表長短不一。我們透過移去百分比限制，交由 Grid stretch 自動對齊拉伸，完美消除了這個底部空隙通病。

🔒 管理者獨立路由存取權限

將管理者設定檔移至 admin.html 會導致設定檔暴露的潛在危險。我們在 Python 伺服器 runtime_server.py 路由註冊中，將該靜態檔案與系統 cookie 密碼過濾網徹底整合，保證未驗證的連線一律被攔截並導向認證頁，保障企業內網資訊安全。
